# 前台中英文切换设计

## 目标

为婚礼网站前台增加中文与英文切换，同时保持现有网址、视觉风格和页面结构不变。

- 默认显示中文。
- 英文姓名显示为 `Hao Wu & Lu Wang`。
- 语言切换只影响前台，后台管理继续使用中文。
- 当前浏览会话内跨页面和刷新后保持语言；关闭浏览器后恢复默认中文。
- 英文文案由后台独立维护，不调用自动翻译服务。
- RSVP、祝福留言等宾客提交内容始终按原文展示。

## 方案选择

采用“会话 Cookie + 服务端双语渲染”。

语言切换接口写入无 `Expires` 和 `Max-Age` 的会话 Cookie。切换成功后刷新当前 Next.js 路由，由服务端按 Cookie 读取对应语言内容。这样可以保持同一网址、避免刷新时先出现中文，也不需要把所有页面改成纯客户端渲染。

不采用以下方案：

- 纯前端状态：刷新时可能闪现错误语言，且会扩大客户端组件范围。
- `?lang=en` 参数或 `/en` 路由：便于分享固定语言，但会改变网址，不符合已确认需求。

## 语言状态

定义两个受支持语言：

```ts
type Locale = "zh" | "en";
```

- Cookie 名称使用 `wedding_locale`。
- 缺少 Cookie、值非法或请求来自新会话时均回退到 `zh`。
- Cookie 使用 `Path=/`、`SameSite=Lax`，生产环境使用 `Secure`。
- 切换接口只接受同源请求和 `zh`、`en` 两个值。
- 切换后保持当前 pathname，不跳回首页。
- 前台页面根据语言设置正确的 `lang` 语义；后台内容和语义保持中文。

## 切换控件

采用已选定的 B 方案：顶部“中文 / EN”分段切换。

- 当前语言使用香槟金实底，另一语言保持透明。
- 手机和桌面使用同一组件和交互。
- 控件放在顶部导航右侧，与菜单按钮并列。
- 手机端隐藏当前未启用的音乐按钮，避免顶部拥挤。
- 切换期间禁用按钮，防止重复请求。
- 控件提供明确的 `aria-label`、当前状态和键盘焦点样式。
- 移动菜单中不重复放置第二套语言控件。

## 内容模型

保留现有 `SiteContent` 页面消费结构，但为每种语言生成一份完整内容。

```ts
type LocalizedSiteContent = Record<Locale, SiteContent>;
```

项目内提供完整的中文和英文默认内容。英文默认内容不是运行时翻译，包含：

- 首页、导航、页脚和倒计时标签。
- 爱情故事。
- 婚礼信息与当天流程。
- 宾客指南、交通、住宿、天气、旅游和美食推荐。
- RSVP、留言簿、相册、回顾页面的标题、说明、表单标签、状态和错误提示。
- 图片替代文本和无障碍标签。

日期、图片、地图链接、酒店链接、功能开关等非语言信息只维护一份并共享。

## 网站设置兼容

现有中文设置键和数据保持不变，例如：

- `site.identity`
- `site.hero`
- `site.wedding`
- `site.story`
- `site.guide`
- `site.navigation`

英文内容使用对应的独立键，例如：

- `site.identity.en`
- `site.hero.en`
- `site.wedding.en`
- `site.story.en`
- `site.guide.en`
- `site.navigation.en`
- `site.travel.en`
- `site.food.en`
- `site.transport.en`
- `site.memories.en`

共享设置继续使用现有键，包括婚礼日期、主视觉图片、地图和酒店链接、相册访问控制、回顾页开关和音乐配置。

内容读取流程：

1. 读取语言对应的项目默认内容。
2. 合并共享设置。
3. 中文合并现有中文设置键；英文合并 `.en` 设置键。
4. 英文字段未填写时回退到项目内置英文默认值，不回退为中文，也不显示空白。

该方案不要求迁移现有 `site_settings` 表结构，只扩展允许保存的 key。

## 相册内容

照片和视频文件、排序、公开状态、集合类型保持共用。为 `photos` 表增加可空字段：

```sql
title_en text
description_en text
```

- 中文标题和描述继续使用现有 `title`、`description`。
- 英文界面优先显示 `title_en`、`description_en`。
- 英文字段为空时使用简短的内置英文兜底，不显示中文标题。
- 分类内部继续保存稳定值，界面标签由语言字典转换，避免为同一照片维护两套分类逻辑。

## 后台编辑

后台界面保持中文。

“网站设置”增加“中文内容 / English Content”页签，并按以下区块编辑：

- 新人姓名和首页文案。
- 婚礼地点、时间说明和当天流程。
- 爱情故事。
- 宾客指南。
- 旅游与美食推荐。
- 交通信息和假期致歉说明。
- 回顾页面文案。

共享字段单独显示，不在两个语言页签中重复：

- 婚礼日期。
- 图片和媒体地址。
- 地图与酒店链接。
- 相册访问控制。
- 音乐和页面开关。

照片管理增加中文、英文标题和描述输入框。保存英文内容不会覆盖中文内容，反之亦然。

## 前台页面行为

以下页面全部支持同步切换：

- 首页。
- 我们的故事。
- 婚礼信息。
- 宾客指南。
- RSVP。
- 婚纱照相册。
- 祝福留言。
- 婚礼回顾。

切换语言后：

- 当前页面位置和路由保持不变。
- 页面导航、按钮、正文、表单和反馈状态使用同一种语言。
- RSVP 和留言 API 数据结构保持不变。
- 已公开留言、姓名和宾客输入内容保持原文。
- 地名英文采用清晰的拼音或官方英文名称，例如 `Taixing`、`Taizhou`。
- 金额继续使用人民币符号和单位，并用英文说明，例如 `Approx. CNY 60`。

## 元数据与可访问性

- 页面标题和描述根据当前语言生成。
- 前台语言切换后更新文档语言语义。
- 表单标签、验证提示、成功提示和图片替代文本均本地化。
- 日期机器值保持不变，显示文案按语言格式化。
- 切换控件满足键盘操作、焦点可见和屏幕阅读器状态播报。

## 错误处理

- 语言 Cookie 非法时静默回退中文。
- 切换请求失败时保持当前语言并恢复按钮可用状态。
- 英文后台字段缺失时使用内置英文默认值。
- 共享设置异常时沿用现有默认内容，避免阻断页面。
- 不把 Cookie、数据库配置或服务端密钥暴露到前端。

## 测试

### 单元与组件测试

- 默认语言为中文。
- 合法和非法 Cookie 的语言解析。
- 中文、英文设置分别合并且互不覆盖。
- 英文字段缺失时使用英文默认内容。
- 语言切换控件状态、禁用状态和可访问性。
- RSVP、留言和相册组件在两种语言下显示正确标签。

### API 与数据库测试

- 语言接口只接受同源的 `zh`、`en` 请求。
- Cookie 为会话 Cookie，不包含持久化过期时间。
- 管理设置 API 接受新增英文键并拒绝未知键。
- 照片英文标题和描述能够创建、更新和读取。

### 端到端测试

- 新会话首页默认中文。
- 切换英文后当前页面变为英文。
- 英文状态在刷新和跨页面导航后保持。
- 新浏览器会话恢复中文。
- 两种语言下 RSVP 和祝福留言均可提交。
- 宾客留言在英文页面保持原文。
- 后台保存英文内容不会覆盖中文内容。
- 手机和桌面端切换控件不遮挡导航、菜单或主视觉。

### 完整验证

运行单元测试、Lint、TypeScript 检查、生产构建和 Playwright 端到端测试，并在手机与桌面视口检查中文和英文首页。
