# RSVP、宾客指南与首页层级调整设计

## 目标

精简宾客回复信息，弱化对宾客的着装要求，补充更实用的交通说明，并增强首页内容标题的视觉层级。

## 交互与内容

- RSVP 不再询问是否携带儿童和饮食忌口。
- 停车需求继续保留，供新人统计车辆数量。
- 后台宾客列表、概览统计和 CSV 导出同步移除儿童与忌口信息。
- “着装建议”改为“天气提醒”，只建议根据天气增减衣物、雨天备伞，不规定颜色或着装形式。
- 公共交通说明覆盖泰州站、泰兴汽车客运站和打车导航，不承诺固定班次或耗时。
- 停车说明只保留酒店提供免费现场停车。
- 首页三个主要中文标题在手机和电脑端各增大一级，封面姓名尺寸保持不变。

## 数据兼容

应用层的 `Guest` 和 RSVP 请求协议移除 `hasChildren` 与
`dietaryRestrictions`。Supabase 已有列暂时保留，写入时使用 `false` 和 `null`
清除旧值，避免破坏现有数据库和迁移历史。

## 验证

- 单元测试覆盖精简后的 RSVP 协议、后台统计和 CSV 表头。
- Playwright 验证 RSVP 页面不出现已移除字段，并保留停车需求。
- 手机与桌面截图验证首页标题字号和布局。
- 运行 lint、TypeScript、完整测试和生产构建。
